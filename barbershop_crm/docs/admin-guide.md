## 1. Вход в систему
Введите логин и пароль учётной записи администратора или менеджера.
| Поле | Описание |
|------|----------|
| Логин | Имя пользователя в системе |
| Пароль | Минимум 8 символов |
После входа вы попадаете на главный дашборд.
## 2. Главный дашборд
Дашборд (`/`) отображает сводную информацию за текущий день:
| Карточка | Описание |
|----------|----------|
| Записей сегодня | Общее число записей на текущую дату |
| Подтверждено | Записи со статусом «Подтверждён» |
| Выручка сегодня | Сумма завершённых записей (₽) |
| Активных мастеров | Число мастеров, работающих сегодня |
Ниже — блок «Ближайшие записи»: таблица с мастером, клиентом, услугой и временем
на сегодня.
## 3. Управление записями
Раздел `/appointments/` — список всех записей клиентов.
### 3.1. Фильтры и поиск
| Фильтр | Описание |
|--------|----------|
| Дата | Диапазон дат или конкретный день |
| Мастер | Фильтр по конкретному мастеру |
| Статус | pending / confirmed / in_progress / done / cancelled / no_show |
| Поиск | По имени или телефону клиента |
### 3.2. Статусы записей
| Статус | Значение |
|--------|----------|
| `pending` | Ожидает подтверждения |
| `confirmed` | Подтверждена |
| `in_progress` | Клиент в кресле |
| `done` | Завершена |
| `cancelled` | Отменена |
| `no_show` | Клиент не пришёл |
### 3.3. Создание записи вручную
Нажмите **«Новая запись»** (`/appointments/new/`).
| Поле | Обязательное | Описание |
|------|:---:|----------|
| Мастер | ✅ | Выбор из активных мастеров |
| Услуга | ✅ | Услуги выбранного мастера |
| Дата и время | ✅ | Дата + слот из расписания |
| Фамилия клиента | ✅ | |
| Имя клиента | ✅ | |
| Телефон | ✅ | Формат: +7 (999) 123-45-67 |
| Комментарий | ❌ | Пожелания клиента |
### 3.4. Редактирование и перенос
Откройте запись (`/appointments/<id>/`) → нажмите **«Редактировать»**. Либо
перетащите карточку в календаре на новый слот — перенос произойдёт автоматически.

## 4. Календарь
Страница `/appointments/calendar/` — визуальный обзор рабочего дня.
### 4.1. Режимы отображения
| Кнопка | Режим |
|--------|-------|
| День | Временная шкала по мастерам (Resource Timeline Day) |
| Неделя | Resource Timeline Week |
| Список | Список записей на день |
### 4.2. Действия в календаре
- **Перетаскивание** карточки — перенос записи на другой слот или к другому мастеру
- **Растягивание** карточки за правый край — изменение длительности
- **Клик** по карточке — всплывающая панель с деталями и кнопками смены статуса
### 4.3. Цветовая кодировка статусов
| Цвет | Статус |
|------|--------|
| Серый | pending |
| Жёлтый | confirmed |
| Синий | in_progress |
| Зелёный | done |
| Красный | cancelled |
| Фиолетовый | no_show |

## 5. Управление мастерами
Раздел `/staff/masters/`.
### 5.1. Создание мастера
Нажмите **«Добавить мастера»**.
| Поле | Описание |
|------|----------|
| Отображаемое имя | Имя, которое видит клиент на странице записи |
| Специализация | Например: «Барбер», «Стилист» |
| Фото | Портрет мастера (JPEG/PNG) |
| Услуги | Список услуг, которые выполняет мастер |
| Активен | Видим ли мастер на странице записи |
### 5.2. Рабочее расписание
После создания мастера откройте его карточку → вкладка **«Расписание»**.
| День | Значение по умолчанию |
|------|-----------------------|
| Понедельник — Суббота | 10:00 – 20:00 |
| Воскресенье | Выходной |
Для каждого дня можно задать произвольное время или отметить как выходной.
### 5.3. Привязка к учётной записи
Мастер может иметь учётную запись для входа в CRM. Привязка происходит при
создании пользователя с ролью **«Мастер»** (см. раздел 7).

## 6. Управление услугами
Раздел `/services/`.
### 6.1. Категории услуг
Услуги объединены в категории (`/services/categories/`).
| Поле | Описание |
|------|----------|
| Название | Например: «Стрижки», «Уход за бородой» |
| Иконка | CSS-класс иконки (опционально) |
| Порядок | Число — определяет порядок отображения |
### 6.2. Создание услуги
| Поле | Описание |
|------|----------|
| Категория | Родительская категория |
| Название | Наименование услуги |
| Описание | Краткое описание (опционально) |
| Длительность | В минутах (например: `60`) |
| Стоимость | В рублях (например: `800.00`) |
| Активна | Отображается ли услуга клиентам |
Пример отображения длительности:
```python
# 30 минут → "30 мин"
# 90 минут → "1ч 30мин"
# 120 минут → "2ч"
```

## 7. Управление пользователями

Раздел `/accounts/users/`.

### 7.1. Роли

| Роль | Доступ |
|------|--------|
| `admin` | Полный доступ ко всем разделам |
| `manager` | Записи, календарь, справочники |
| `master` | Только свои записи (ограниченный вид) |

### 7.2. Создание пользователя

Нажмите **«Добавить пользователя»** (`/accounts/users/new/`).

| Поле | Описание |
|------|----------|
| Логин | Уникальное имя для входа |
| Имя / Фамилия | Отображаемое имя |
| Email | Электронная почта |
| Пароль | Минимум 8 символов |
| Роль | admin / manager / master |

> **Важно:** при выборе роли **«Мастер»** система автоматически:
> - создаёт профиль мастера (`staff_master`)
> - назначает все активные услуги
> - добавляет расписание Пн–Сб 10:00–20:00

### 7.3. Сброс пароля

Откройте пользователя → нажмите **«Сбросить пароль»** → введите новый пароль
дважды.

## 8. Аналитика

Раздел `/analytics/` — статистика за выбранный период.

| Блок | Описание |
|------|----------|
| Выручка по дням | График поступлений (₽) |
| Топ услуг | Наиболее популярные услуги |
| Топ мастеров | По числу завершённых записей |
| Конверсия | Соотношение pending → done |

Для экспорта данных нажмите **«Экспорт CSV»**.
