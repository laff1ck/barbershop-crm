Развёртывание и сопровождение системы

Глава 3.6 · Deploy & Maintenance
🐳 Docker Compose 01
  • Контейнеризация всех компонентов системы
  • Изолированные сервисы: web / db / nginx
  • docker-compose up --build — запуск в одну команду
  • Воспроизводимость среды на любом Linux-хосте
  • Переменные окружения через .env-файл
⚡ Gunicorn + Nginx 02
  • Gunicorn: WSGI-сервер для Django (4 workers)
  • Nginx: обратный прокси + раздача статики
  • HTTPS-терминация через SSL-сертификат
  • Статические файлы отдаются без Django
  • Соответствует нефункц. требованию безопасности
🗄️ PostgreSQL + миграции 03
  • База данных в отдельном Docker-контейнере
  • Том (volume) для постоянного хранения данных
  • python manage.py migrate — применение схемы
  • Django Migrations: версионирование изменений
  • Автоматическое создание схемы при первом запуске
🔧 Сопровождение 04
  • Журналы Gunicorn и Nginx через docker logs
  • Резервное копирование БД: pg_dump по расписанию
  • Обновление: git pull → migrate → restart
  • Команда seed_data для начальных данных
  • Django Admin — управление пользователями и ролями
